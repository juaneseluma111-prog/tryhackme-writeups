## Objective

to think like an attacker to better defend systems.

---

## Key Concepts Learned

### What is Offensive Security
- Simulating attacks to find vulnerabilities before real attackers do
- Legal and authorized testing of systems
- Red Team vs Blue Team roles

### Basic Reconnaissance
- Passive recon — gathering info without touching the target
- Active recon — directly interacting with the target
- OSINT — Open Source Intelligence gathering

### First Hands-on Tool — Gobuster
```bash
gobuster dir -u http://target.com -w /usr/share/wordlists/dirb/common.txt
```
Used to discover hidden directories on web servers.

---

## Key Learnings
- Offensive security is about finding weaknesses to fix them
- Reconnaissance is always the first phase of any engagement
- Even basic tools like Gobuster can reveal critical information
